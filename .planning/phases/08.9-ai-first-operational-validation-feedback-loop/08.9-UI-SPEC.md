# UI Spec - 08.9 AI First Review Feedback

## Principio

A tela deve continuar sendo uma mesa de revisao, nao um dashboard pesado. O operador precisa revisar, ajustar e decidir rapidamente.

## Mudancas

- Inspector ganha bloco `Feedback do revisor`.
- Textarea editavel com o rascunho atual.
- Campo curto de nota/motivo.
- CTAs:
  - `Aprovar`
  - `Salvar edicao`
  - `Preparar na Kommo`
  - `Escalar`
  - `Rejeitar`
- Se o item ja tiver feedback, mostrar:
  - decisao
  - data
  - observacao
  - se houve edicao.

## Estados

- `approved`: rascunho aprovado para handoff humano.
- `edited`: rascunho foi ajustado antes do handoff.
- `rejected`: rascunho nao deve ser usado.
- `escalated`: exige decisao humana fora do fluxo simples.

## Linguagem

- Evitar prometer envio automatico.
- Usar termos: `rascunho`, `handoff`, `revisao humana`, `bloqueio`, `escalado`.
