# 04.40 Plan

## Objetivo

Substituir o modelo atual de sessao por algo menos fragil e fechar a borda publica de producao com defaults mais seguros.

## Escopo

1. Redesenhar armazenamento de refresh/access token para reduzir blast radius de XSS
2. Adicionar baseline de headers de seguranca e CSP
3. Falhar build/producao em caso de env ausente ou default perigoso
4. Revisar websocket e bootstrap do frontend sob a nova estrategia de sessao

## Critérios de aceite

- sessao publicada tem blast radius menor do que `localStorage` puro
- frontend e backend servem headers de seguranca coerentes
- producao nao sobe com defaults inseguros como `localhost`
- testes cobrem bootstrap de API, auth e borda
