## Objetivo

Remover sinais cenograficos ou excessivamente otimistas das superficies operacionais do piloto. A equipe nao deve ver numero, badge ou card sugerindo sucesso operacional quando a base real ainda nao sustenta aquela leitura.

## Entregue

- Removido fallback de email `owner@demo.local` no layout autenticado.
- Dashboard executivo agora mostra `Sem base` para receita quando nao ha MRR real.
- Dashboard executivo agora mostra `Sem base` para churn quando nao ha membros/serie real.
- Card de ROI das automacoes deixou de afirmar que automacoes estao ativas ou que ha tempo medio de reengajamento sem resultados confirmados.
- Regressao de frontend adicionada para empty state honesto de ROI e KPIs sem base.

## Validacao

- `cmd /c npm.cmd run test -- --run src/test/DashboardLovable.test.tsx src/test/RoiSummaryCard.test.tsx`
- `cmd /c npm.cmd run build`

## Decisao

Indicadores comerciais, financeiros, ROI e execucao de automacoes continuam visiveis, mas devem ser apresentados como `Sem base` ou empty state operacional enquanto nao houver dado real suficiente.
