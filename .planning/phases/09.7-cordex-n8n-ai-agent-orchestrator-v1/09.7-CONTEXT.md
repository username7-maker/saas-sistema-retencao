# 09.7 - Contexto

## Objetivo

Corrigir a V1 do Cordex n8n AI Agent para ser WhatsApp-first. O backend continua dono do webhook Evolution API e o n8n vira o cerebro de decisao acionado por payload normalizado.

## Escopo

- Pacote `/n8n` com router WhatsApp, agente interno, agente externo, send reply via backend, tools, prompts, SQL e documentacao.
- Backend chamando n8n a partir do webhook WhatsApp existente.
- Backend preservando tenant, status da instancia, logs, rate limit, parsing e fallback legado.
- Drafts inativos no n8n para revisao.
- Postgres como persistencia de logs, memoria, permissoes, tarefas, eventos WhatsApp e RAG.
- OpenAI como LLM/embeddings padrao.
- Approval gate por WhatsApp para acoes sensiveis.

## Fora de escopo

- Ativacao em producao.
- Credenciais reais.
- Envio real pelo agente em sandbox.
- n8n chamando Evolution diretamente.
- Robo externo irrestrito para leads/alunos.

## Decisoes

- O canal principal da V1 e WhatsApp.
- O backend chama n8n por `N8N_WHATSAPP_AGENT_WEBHOOK_URL`.
- O n8n retorna decisao; o backend envia via `send_whatsapp_sync` quando ativo.
- Numeros allowlisted entram como agente interno.
- Leads/alunos entram no agente externo controlado.
- Fluxo externo responde apenas ao mesmo remetente.
- Telegram foi removido do caminho principal.
