# Phase 4.31 UI Spec

## Intent

Transformar o fluxo de bioimpedancia em uma experiencia confiavel para piloto, deixando claro:

- o que foi lido automaticamente
- o que foi inferido por IA
- o que esta pendente de revisao manual
- se o Actuar esta ativo, parcial ou desabilitado

## UX principles

1. Nada de CTA enganoso:
   - se o Actuar estiver desabilitado, o estado principal nao pode parecer `pendente de sync automatico`
2. Origem sempre visivel:
   - OCR local
   - IA assistida
   - fallback/manual
3. Manual assistido e um fluxo legitimo:
   - nao deve parecer erro escondido
4. Campos nao enviados ao Actuar precisam aparecer como tal:
   - nao como detalhe tecnico escondido

## Required UI states

### State A - OCR local somente

- mostrar que a leitura veio do OCR local
- explicar que revisao manual continua obrigatoria
- esconder linguagem que sugira IA ativa

### State B - IA assistida ativa

- mostrar que houve leitura assistida
- marcar campos substituidos/confirmados pela IA
- manter warnings e revisao onde necessario

### State C - Actuar desabilitado

- mostrar badge/estado explicito de `Sync Actuar desabilitado`
- CTA principal vira:
  - salvar localmente
  - copiar resumo manual
- nao deixar `sync pending` como estado dominante

### State D - Actuar parcial/manual

- exibir claramente:
  - campos criticos
  - campos nao suportados
  - motivo do fallback manual
- confirmar sync manual com auditoria continua permitido

## Affected surfaces

- `MemberBodyCompositionTab`
- resumo de status de avaliacao corporal
- mensagens de toast e labels de origem/status

## Acceptance cues

- um professor consegue dizer em 5 segundos se a leitura esta:
  - automatica
  - assistida
  - manual
- um gestor consegue dizer em 5 segundos se o Actuar:
  - esta desligado
  - esta em fallback manual
  - sincronizou de verdade
