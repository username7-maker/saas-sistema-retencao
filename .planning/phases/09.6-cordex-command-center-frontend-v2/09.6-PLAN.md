# 09.6 - Plano

## Resultado esperado
O sistema deve parecer um centro de comando operacional premium, usando dados reais e mantendo fluxos atuais.

## Etapas
1. Criar componentes premium reutilizáveis em `src/components/ui2/command`.
2. Evoluir tokens em `lovable-theme.css`, `index.css`, `tailwind.config.js` e `brand.ts`.
3. Refatorar `LovableLayout.tsx` para App Shell Cordex Command Center.
4. Refatorar `DashboardLovable.tsx` com hero executivo, KPIs, mapa operacional, briefing, matriz de risco e fila de ações.
5. Aplicar linguagem visual nos dashboards operacional, comercial, financeiro e retenção sem alterar contratos.
6. Rodar lint, build, testes e Spec Kit.

## Não escopo
- Redesign profundo de Membros, Tasks, Avaliações, CRM, Settings e Relatórios.
- Mudança de backend.
- Mudança de API, cookies, localStorage, RBAC ou permissões.
- Dados mockados em páginas que já usam hooks reais.

## Critérios de aceite
- Dashboard executivo usa dados reais ou empty states explícitos.
- Sidebar/topbar parecem premium e preservam navegação.
- Componentes base são reutilizáveis e tipados.
- Dashboards principais carregam sem quebrar hooks.
- Build frontend passa.
