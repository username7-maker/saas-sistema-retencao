# 09.6 - Contexto

## Objetivo
Elevar o frontend do Cordex Gym OS para uma linguagem visual premium de Command Center: dark enterprise, alta densidade de informação, hierarquia clara e sensação de SaaS high-ticket.

## Escopo desta entrega
- Design system visual e componentes base.
- App shell global: sidebar, topbar, busca, status e navegação.
- Dashboards principais: executivo, operacional, comercial, financeiro e retenção.
- Sem alterações de backend, contratos de API, autenticação, RBAC, storage keys ou rotas.

## Contexto existente
- O rebrand visual para Cordex já existe em fases anteriores.
- `lovable-*` continua sendo compatibilidade técnica.
- O frontend usa React 18, TypeScript, Vite, Tailwind, React Query e Recharts.
- Existem páginas operacionais grandes que devem herdar tokens nesta fase, mas não serão redesenhadas profundamente agora.

## Decisão de produto
Cordex Command Center passa a ser o padrão visual premium. A implementação será em ondas para preservar a operação real do piloto.
