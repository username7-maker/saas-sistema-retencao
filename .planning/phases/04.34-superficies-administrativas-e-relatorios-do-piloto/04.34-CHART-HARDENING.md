# 04.34 Chart Hardening

## Trigger

Durante o piloto, o usuario reportou que alguns graficos "pareciam quebrados" em superficies tipo dashboard.

## Diagnostico confirmado

- Os dashboards podiam receber payload estruturalmente valido, mas sem base historica util.
- Nessas condicoes, a UI ainda tentava desenhar graficos zerados ou vazios como se fossem insight real.
- No piloto atual, isso aparecia com mais forca em:
  - dashboard executivo
  - dashboard financeiro
  - dashboard comercial

## Ajuste aplicado

- helper compartilhado `components/charts/chartState.ts` passou a classificar:
  - serie vazia
  - serie zerada
  - serie flat
- `DashboardLovable` agora mostra estado honesto quando churn e NPS nao tem base historica util
- `FinancialDashboardPage` agora troca curva por estado vazio contextual quando receita historica esta sem base
- `CommercialDashboardPage` agora troca funil/tabela por estados vazios contextuais quando o pipeline ainda nao tem base suficiente
- `LineSeriesChart`, `BarSeriesChart` e `PieRiskChart` foram endurecidos para nao renderizar graficos mortos por padrao

## Validacao

- `npm run test -- src/test/chartState.test.ts src/test/DashboardLovable.test.tsx` -> verde
- `npm run build` -> verde
- frontend republicado na Vercel e re-aliasado em `https://saas-frontend-pearl.vercel.app`

## Leitura honesta

Este hotfix nao "inventou dado". Ele melhorou a honestidade do produto:

- quando houver serie util, o grafico continua
- quando nao houver base historica suficiente, a interface assume isso explicitamente
