# 04.34 UI Spec

## Objetivo visual e de UX

- A área administrativa precisa parecer estável e utilizável em produção.
- Estados vazios devem ser explícitos, sem parecer erro.
- Ações de salvar/baixar precisam devolver feedback claro.
- Perfil e equipe devem permitir personalização mínima sem inflar a interface.

## Contratos de superfície

### Notificações
- Deve ter título correto da seção.
- Quando vazio, mostrar mensagem clara e call-to-action simples.
- Quando der erro, o erro deve refletir problema real, não bug de URL.

### Metas
- Formulário deve salvar com feedback confiável.
- Estado vazio deve orientar o primeiro cadastro.

### Relatórios
- Cartões devem explicar o uso.
- “Baixar PDF” precisa funcionar de verdade para os relatórios suportados.
- Disparo mensal precisa usar a mesma base sem quebrar.

### NPS
- Com zero respostas, mostrar vazio de negócio: o que significa e o próximo passo.
- CTA de disparo segue visível.

### Perfil e usuários
- Exibir avatar com fallback por iniciais.
- Exibir cargo quando existir.
- Permitir editar nome, cargo e avatar por URL.
- Owner pode editar papel da equipe; manager não.
