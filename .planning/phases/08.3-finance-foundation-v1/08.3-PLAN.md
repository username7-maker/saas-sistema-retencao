# Phase 8.3 Plan - Finance Foundation V1

## Objetivo

Entregar uma fundacao financeira operacional para o piloto, com caixa diario, contas a receber/pagar, inadimplencia, DRE basico, receita em risco e leitura honesta de falta de base.

## Escopo

- Criar entidade tenant-scoped para lancamentos financeiros.
- Permitir registrar:
  - recebiveis;
  - pagaveis;
  - entradas de caixa;
  - saidas de caixa.
- Controlar status operacional:
  - aberto;
  - pago;
  - atrasado;
  - cancelado.
- Expor CRUD protegido por role de gestao.
- Calcular resumo financeiro com:
  - caixa do dia;
  - contas abertas;
  - contas vencidas;
  - inadimplencia;
  - DRE basico;
  - receita em risco;
  - flags de qualidade de dados.
- Integrar o dashboard financeiro ao novo resumo sem remover fallback honesto quando ainda nao houver base financeira suficiente.
- Mostrar no frontend um formulario simples de lancamento e uma leitura executiva mais operacional.

## Fora do escopo

- Integracao bancaria.
- Conciliacao automatica.
- Gateway de pagamento.
- Nota fiscal.
- Parcelamento avancado.
- Centro de custo completo.
- Forecast financeiro sofisticado por churn probabilistico.

## Decisoes

- A fonte financeira real da V1 e manual/importada via lancamento, nao inferida silenciosamente de membros.
- O dashboard pode usar fallback de mensalidade enquanto nao existir lancamento pago, mas deve continuar expondo flags de qualidade.
- O modulo nasce como fundacao para gestao, nao como fluxo de recepcao.
- Financeiro permanece protegido para `owner` e `manager`.

## Validacao

- Testes unitarios do service financeiro.
- Testes de regressao dos cortes T201/T202/T203.
- Build frontend.
- `specify check`.
- Validacao visual posterior no piloto com base real.
