- root cause:
  - o importador nao reutilizava de forma consistente os membros criados/atualizados em memoria quando o match dependia so de nome
- tratamento aplicado:
  - preview e import passaram a usar o mesmo resolvedor de match
  - o lookup em memoria passou a ser atualizado de forma idempotente
  - foi adicionado script de limpeza para os duplicados historicos do piloto
- estado atual:
  - o mesmo XLSX pode ser reprocessado em preview sem reabrir a onda de `1.629` criacoes artificiais
  - a base do piloto foi saneada antes de seguir no dia 0

## Achado resolvido de infraestrutura do dia 0

- root cause:

- tratamento aplicado:

  - API e worker foram redeployados no piloto
- estado atual:

  - a fila de recalc duravel voltou a processar ate `completed`
