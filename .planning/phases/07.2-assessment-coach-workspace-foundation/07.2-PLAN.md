# Phase 7.2 Plan - Assessment + Coach Workspace Foundation

## Goal

Criar o primeiro Coach Workspace staff-first: uma fila tecnica por turno para professores acompanharem avaliacao, bioimpedancia, treino, feedback e reavaliacao sem misturar com retencao/recepcao.

## Scope deste corte

### Backend

- Criar payload semantico `CoachWorkspaceItem`.
- Criar service `coach_workspace_service`.
- Criar rota `GET /api/v1/coach/workspace`.
- Usar fontes reais ja existentes:
  - `Member`
  - `Assessment`
  - `BodyCompositionEvaluation`
  - `Task`
  - `TaskEvent`
  - `MemberTrainingPlan`, quando existir no repo
  - `LeadToMemberIntelligenceContext`, quando aplicavel
- Classificar itens por lane tecnica:
  - `training_delivery`
  - `training_feedback`
  - `reassessment`
  - `assessment_pending`
  - `body_composition_review`
  - `training_adjustment`
  - `technical_attention`
- Respeitar `work_shift` e `work_shift_scope` do professor.
- Managers/owners podem usar `shift=all`.
- Receptionists nao devem receber fila tecnica como padrao.
- Retencao e onboarding operacional seguem fora da fila do professor, salvo quando forem explicitamente etapa tecnica.

### Frontend

- Criar entrada visual `Coach Workspace` dentro do contexto de `Avaliacoes` ou `Tarefas > Professor`.
- Exibir:
  - alunos do turno;
  - etapa tecnica;
  - proxima acao;
  - motivo curto;
  - prazo;
  - ultima avaliacao/bioimpedancia;
  - CTA principal;
  - outcomes rapidos.
- Reaproveitar o visual da Work Queue quando possivel.
- Evitar dashboard grande no primeiro corte.

### Produto

- Manter rotina principal do professor como lista curta de execucao.
- Separar responsabilidades:
  - professor: tecnico;
  - recepcao: onboarding, contato, retencao;
  - gestor: gargalos e turnos.
- Tornar bioimpedancia e avaliacao parte do mesmo ciclo tecnico.

## Fora do corte

- Nova tabela obrigatoria de CoachDecision.
- Criacao automatica de treino.
- IA generativa para prescricao.
- App do aluno.
- Agenda completa.
- Metricas complexas de performance por professor.
- Reescrever Perfil 360.

## Deliverables

1. Spec Kit `018-assessment-coach-workspace-foundation`.
2. GSD phase `07.2-assessment-coach-workspace-foundation`.
3. Obsidian atualizado com decisao e open loops.
4. Backend service/route para workspace tecnico.
5. Frontend V1 ou adaptacao da fila do professor para consumir o payload.
6. Testes focados de filtro por turno, role e lanes.
7. Validacao piloto com pelo menos um professor por turno.

## Implementation waves

### Wave 1 - Contrato e payload

- Definir schema `CoachWorkspaceItem`.
- Implementar service read-only.
- Criar endpoint tenant-scoped.
- Criar testes de role/turno/lane.

### Wave 2 - UI operacional

- Criar/ajustar tela de fila tecnica.
- Mostrar apenas campos acionaveis.
- Reaproveitar outcomes tecnicos existentes.
- Garantir empty/loading/error honestos.

### Wave 3 - Piloto e ajuste fino

- Testar com professor manha, tarde e noite/madrugada.
- Validar se tarefas de recepcao nao aparecem como professor.
- Medir se professor entende proxima acao em menos de 20 segundos.

## Success criteria

- Professor ve apenas fila tecnica do seu turno por padrao.
- Manager/owner consegue revisar todos os turnos.
- Primeira avaliacao operacional nao cai indevidamente para professor.
- Tasks D+8, D+14 e D+90 aparecem como tecnicas quando a janela chega.
- Bioimpedancia recente entra como ciclo tecnico quando exigir revisao/feedback.
- Retencao comum nao polui a fila tecnica.
- O sistema deixa claro quando nao ha dados suficientes.
