# Phase 7.2 Validation - Assessment + Coach Workspace Foundation

## Validacao planejada

### Backend

- Professor com `work_shift=morning` ve apenas alunos/itens tecnicos de manha.
- Professor com escopo noite+madrugada ve `evening` e `overnight`.
- Owner/manager com `shift=all` ve todos os turnos.
- Receptionist nao recebe fila tecnica como padrao.
- Primeira avaliacao operacional fica fora da lane tecnica do professor.
- D+8/D+14/D+90 entram nas lanes corretas quando a janela chega.
- Bioimpedancia recente aparece como ciclo tecnico quando aplicavel.
- Retencao comum nao aparece na fila tecnica.
- Tenant isolation preservado.

### Frontend

- Fila tecnica renderiza card curto e CTA principal.
- Empty state de login sem turno aparece claramente.
- Detalhe mostra evidencias e outcomes tecnicos.
- Dados financeiros/comerciais nao aparecem para professor.
- `Abrir Perfil 360` funciona como contexto, nao como dependencia para agir.

### Piloto

- Testar com professor manha, tarde e noite/madrugada.
- Executar 10 itens tecnicos reais.
- Confirmar se cada item e entendido em menos de 20 segundos.
- Confirmar se recepcao/gestor nao precisam explicar manualmente a fila.

## Evidencias

- `GET /api/v1/coach/workspace` implementado como endpoint tenant-scoped para `owner`, `manager` e `trainer`.
- Service read-only usa a Work Queue existente com `domain=trainer`, preservando turno, visibilidade futura e filtro de tasks tecnicas.
- UI V1 criada em `Tarefas > Modo execucao` para logins `trainer`, com lanes, evidencias, contexto e outcomes tecnicos.
- UI V1 tambem aparece para `owner/manager` como aba `Professor`, com filtro `Todos os turnos`.

## Ajustes feitos durante validacao

- O teste frontend antigo ainda esperava a fila tecnica via `workQueueService.listItems`.
- A 7.2 criou o contrato correto `coachWorkspaceService.getWorkspace`, portanto o teste foi atualizado para validar o endpoint novo e a linguagem atual da UI.
- Esse ajuste confirma que trainer abre a fila tecnica por padrao com `state=do_now`, `shift=my_shift`, `page=1` e `page_size=25`.
