# Phase 4.41 Plan - Handoff seguro para Kommo

## Goal

Entregar uma primeira integracao segura e usavel com a Kommo, sem disputar o numero oficial da academia no AI GYM OS.

## Plan

1. Adicionar configuracao por academia
- campos no model `Gym`
- migration
- schemas/servicos/rotas para `GET/PUT/test-connection`

2. Criar servico de handoff
- validar prontidao da Kommo
- criar lead/contato e tarefa
- registrar ids locais por membro

3. Conectar o motor de automacoes
- adicionar `send_to_kommo`
- manter comportamento atual das demais acoes

4. Conectar a bioimpedancia
- botao manual `Enviar para Kommo`
- resumo do aluno + contexto operacional + link do exame

5. Validar
- testes backend focados
- lint/build do frontend
- registrar limitacoes honestas para o piloto

## Exit criteria

- owner/manager conseguem salvar e testar configuracao da Kommo
- automacao consegue entregar handoff para a Kommo
- bioimpedancia consegue gerar handoff manual
- nenhuma automacao/fluxo atual do piloto quebra no processo
