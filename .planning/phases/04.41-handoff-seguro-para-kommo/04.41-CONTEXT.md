# Phase 4.41 Context - Handoff seguro para Kommo

## Why this phase exists

A academia ja usa a Kommo como canal operacional. Manter o AI GYM OS conectado por QR no mesmo numero oficial aumenta risco de sessao instavel, desconexao e perda de confianca logo no piloto.

O desenho correto para esta fase e:

- AI GYM OS decide
- Kommo executa

## Scope

- configuracao da Kommo por academia dentro de `Settings`
- teste de conexao para owner/manager
- nova acao de automacao `send_to_kommo`
- handoff manual da bioimpedancia para a Kommo
- persistencia local do vinculo entre membro e ids Kommo para evitar duplicacao facil

## Explicit non-goals in this first slice

- assumir o canal de WhatsApp oficial dentro do AI GYM OS
- mandar mensagem pelo numero oficial via API propria do AI GYM OS
- depender de Salesbot/Chats API da Kommo nesta primeira entrega
- anexar PDF diretamente na Kommo sem validar primeiro o fluxo seguro da conta real

## Operational framing

Esta fase entrega o caminho seguro para o piloto desta semana:

- AI GYM OS continua sendo o cerebro
- Kommo vira o lugar onde a equipe recebe e opera a acao
