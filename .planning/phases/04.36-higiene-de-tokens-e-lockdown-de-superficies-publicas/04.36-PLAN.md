# 04.36 Plan

## Objetivo

Fechar os riscos P0 de segredos em URL e abuso publico antes de qualquer exposicao maior do piloto.

## Escopo

1. Remover token de query string de reset de senha
2. Remover ou substituir token de query string do websocket por contrato mais seguro
3. Remover suporte a token via query string no webhook publico
4. Endurecer `/health/ready` para nao expor detalhes internos
5. Falhar o startup em producao quando o rate limit real nao estiver habilitado
6. Revisar contratos publicos mais sensiveis e exigir controles minimos de abuso

## Critérios de aceite

- nenhum endpoint sensivel depende de segredo em query string
- readiness publica nao vaza erro de banco/cache
- producao nao sobe sem rate limiting real
- fluxos publicos mais perigosos ficam menos abusaveis do que hoje
- testes cobrindo reset, websocket, webhook e readiness
