# Phase 4.42 Context - Actuar Bridge local

## Why this phase exists

O modo atual de sync `assisted_rpa` roda no worker com um navegador isolado. Isso nao aproveita a aba do Actuar que ja esta aberta e logada no computador do professor, e cria um desenho fraco para o piloto.

O metodo certo para o cenario da academia e:

- AI GYM OS decide
- uma estacao local faz a automacao
- a estacao usa a sessao ja aberta do Actuar no navegador do operador

## Scope

- novo modo de sync `local_bridge`
- pareamento seguro de uma estacao local por academia
- heartbeat e status da estacao
- fila dedicada para a ponte local reivindicar jobs do Actuar
- callback seguro de sucesso/falha da automacao
- scaffold do app local `Actuar Bridge`

## Explicit non-goals in this first slice

- prometer que a automacao server-side do worker agora usa a aba do operador
- depender de API oficial do Actuar
- depender de webhook do Actuar
- entregar extensao de navegador completa ja nesta primeira fatia
- remover o fallback `csv_export/manual assistido`

## Operational framing

Esta fase nao troca o piloto para modo automatico por default.

Ela prepara o caminho correto para ativacao futura:

- uma ou mais estacoes locais pareadas
- sync automatico usando a sessao local
- worker deixando de disputar o mesmo job quando o modo `local_bridge` estiver ativo
