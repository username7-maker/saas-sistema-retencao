# Phase 7.3 Plan - BI Foundation Upgrade

## Objetivo

Transformar a base de BI existente em uma leitura executiva operacional, conectando cohort/LTV/forecast com onboarding, retencao por estagio, execucao da equipe e Autopilot.

## Tarefas

- [x] Estender schemas de BI com onboarding activation, retention stage mix, staff execution, AI First Ops e manager actions.
- [x] Evoluir `dashboard_service.get_bi_foundation_dashboard` sem criar nova tabela.
- [x] Calcular ativacao de onboarding incluindo avaliacao formal e bioimpedancia.
- [x] Calcular mix de estagios de retencao.
- [x] Calcular execucao da equipe por tasks concluidas e vencidas.
- [x] Calcular leitura basica de Autopilot por actions, sucesso, escalamento e aguardando.
- [x] Gerar acoes gerenciais a partir de thresholds operacionais.
- [x] Atualizar dashboard executivo e relatorios com leitura compacta.
- [x] Adicionar testes backend e frontend focados.
- [ ] Validar com gestao da academia se as acoes gerenciais sao legiveis.

## Riscos

- Dados de Autopilot podem estar sem base no piloto inicial. Mitigacao: `data_quality_flags`.
- Tasks antigas podem inflar backlog. Mitigacao: expor backlog como acao de gestao, nao como metrica de sucesso.
- BI pode induzir falsa precisao. Mitigacao: manter labels de base insuficiente.

## Evidencias esperadas

- `GET /api/v1/dashboards/bi-foundation` retorna novas secoes sem quebrar consumidores atuais.
- Dashboard mostra ativacao/execucao/Autopilot e acoes da gestao.
- Reports mostram execucao e Autopilot na base de BI.
