# Phase 7.3 UI Spec - BI Foundation Upgrade

## Principio

A UI deve ser compacta e executiva. A gestao nao precisa de uma tabela enorme; precisa de sinais de decisao e flags honestas.

## Dashboard Executivo

Manter os cinco cards existentes:

- Cohort.
- LTV estimado.
- Forecast.
- Receita em risco.
- Impacto follow-up.

Adicionar bloco operacional:

- Onboarding ativo.
- Tasks concluidas em 7 dias.
- Resolucao do Autopilot, quando houver base.
- Aviso de backlog vencido.

Adicionar bloco de acoes gerenciais:

- Mostrar ate 2 acoes principais.
- Cada acao tem titulo e motivo curto.

Adicionar chips de retencao:

- Mostrar apenas estagios com volume em recuperacao, reativacao, escalacao ou base fria.

## Relatorios

Na Base de BI:

- Manter cohort, LTV, forecast e follow-up.
- Adicionar execucao 7d.
- Adicionar Autopilot.
- Mostrar primeira proxima acao de gestao.

## Estados

- Loading existente continua.
- Sem dados: manter mensagem de indisponibilidade.
- Base insuficiente: usar `data_quality_flags`.
