# Plano 09.2 - Cordex Design System V1

## Objetivo

Dar ao produto uma cara Cordex consistente sem alterar contratos operacionais. A V1 deve ser perceptivel no primeiro acesso: login, sidebar, cards, botoes, badges, configuracoes, work queue e relatorios.

## Entregas

1. Atualizar tokens em `lovable-theme.css` para preto/off-white/azul nucleo.
2. Adicionar `BRAND_COLORS`, `BRAND_SURFACES`, `BRAND_MOTION` e assets Cordex em `brand.ts`.
3. Gerar assets:
   - `cordex-mark-light.png`
   - `cordex-mark-dark.png`
   - `cordex-orbit-watermark.png`
   - `favicon.png`
4. Atualizar componentes base `Button`, `Card`, `Badge`, `Input`, `Select`, `Textarea`, `Tabs`, `Dialog`, `Drawer`.
5. Atualizar login/reset e shell/sidebar para usar mark Cordex e atmosfera orbital.
6. Renomear labels visiveis:
   - `AI Inbox` -> `Central Cordex`
   - `Revisao IA` -> `Revisao Cordex`
   - `Autopilot` -> `Cordex Autopilot`
   - `Personal IA` -> `Cordex Coach`
   - `Video IA` -> `Cordex Motion`
   - `Agente IA` -> `Cordex Agent`
7. Atualizar relatorios de bioimpedancia/premium para acento azul Cordex.
8. Rodar lint/build e listar legados mantidos.

## Criterios de Aceite

- Login e sidebar parecem Cordex, nao template roxo.
- Relatorio de bioimpedancia mostra Cordex como software e ProGym como academia.
- Usuarios existentes continuam logando.
- `lovable-*` continua tecnico e compativel.
- Nenhuma migration, rota, cookie ou token tecnico renomeado.
