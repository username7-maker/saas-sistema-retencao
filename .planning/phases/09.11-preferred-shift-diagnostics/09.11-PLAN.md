# 09.11 - Plano

## Resultado esperado
Quando um item da fila estiver `Sem turno`, o operador deve ver o motivo operacional de forma curta.

## Etapas
1. Registrar Spec Kit/GSD.
2. Criar diagnostico a partir das contagens de check-ins recentes.
3. Enviar diagnostico no contrato da Work Queue.
4. Renderizar motivo no card e no detalhe lateral somente para `Sem turno`.
5. Testar backend e frontend.
6. Publicar piloto.

## Criterios de aceite
- Sem check-in recente/importado fica claro.
- Empate mostra contagem por turno.
- Itens com turno definido nao ganham ruido visual.
- Contrato e aditivo.
