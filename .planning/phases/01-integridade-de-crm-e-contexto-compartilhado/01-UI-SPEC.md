# Phase 1 UI Spec

## CRM Drawer

- Em edicao, a secao principal passa a se chamar `Historico comercial`.
- O historico deve listar eventos com tipo, metadata resumida e data.
- O operador adiciona novas observacoes por um campo de append dedicado.
- O formulario principal continua editando apenas campos escalares e `conversion_handoff`.

## Profile 360

- A secao `Notas internas` continua visivel para roles autorizadas.
- Nenhuma nota local silenciosa deve aparecer como se fosse compartilhada.
- Erro de rede/permissao ao salvar nota precisa ser tratado como erro explicito.
