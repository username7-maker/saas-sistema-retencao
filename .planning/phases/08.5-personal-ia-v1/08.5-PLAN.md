# Phase 8.5 Plan - Personal IA V1

## Objetivo

Criar o Personal IA V1 como assistente tecnico supervisionado pelo professor. Ele deve preparar orientacao personalizada baseada em dados reais do aluno, sem prescrever treino autonomamente e sem responder casos sensiveis.

## Non-goals

- Nao criar personal autonomo.
- Nao criar prescricao automatica de treino.
- Nao criar dieta, suplementacao ou orientacao medica.
- Nao analisar video/movimento.
- Nao enviar mensagem automaticamente para aluno.
- Nao criar app do aluno nesta fase.

## Wave 0 - Contrato seguro

- Criar settings por academia:
  - `personal_ai_enabled=false`;
  - `personal_ai_mode=coach_review`;
  - `personal_ai_auto_send_enabled=false`;
  - `personal_ai_sensitive_escalation_enabled=true`;
  - dominios habilitados: `training_guidance`, `routine_support`, `assessment_explanation`, `body_composition_explanation`.
- Definir termos bloqueados:
  - dor forte, lesao, tontura, falta de ar, emergencia;
  - diagnostico, remedio, suplemento, dieta clinica;
  - pedido medico/fisioterapeuta;
  - cancelamento, reclamacao grave, assedio/agressao.

## Wave 1 - Contexto Personal IA

- Criar payload `PersonalAiContext` a partir de fontes existentes:
  - membro/lifecycle;
  - avaliacao formal mais recente;
  - bioimpedancia mais recente;
  - plano de treino ativo;
  - metas ativas;
  - restricoes;
  - check-ins recentes;
  - tasks tecnicas abertas;
  - TaskEvents tecnicos;
  - consentimento e canal principal.
- Expor endpoint:
  - `GET /api/v1/members/{member_id}/personal-ai/context`.

## Wave 2 - Draft tecnico supervisionado

- Criar service `personal_ai_service.py`.
- Criar rascunho estruturado reusando `AutopilotAction` com `action_type=personal_ai_guidance_draft`.
- Entradas:
  - pergunta do aluno/professor;
  - dominio;
  - contexto canonico;
  - canal desejado.
- Saida:
  - `intent`;
  - `safety_level`;
  - `summary`;
  - `draft_guidance`;
  - `next_action`;
  - `recommended_owner_role`;
  - `blocked_reasons`;
  - `evidence`.
- Endpoint:
  - `POST /api/v1/members/{member_id}/personal-ai/drafts`.

## Wave 3 - Guardrails

- Bloquear rascunho direto se:
  - sem consentimento de comunicacao para envio externo;
  - aluno sem avaliacao/bioimpedancia/plano suficiente;
  - restricao de saude exige professor;
  - pergunta pede dieta, suplemento, medicina ou diagnostico;
  - ha dor/lesao/sintoma;
  - humano respondeu recentemente;
  - caso esta em atendimento Kommo ativo.
- Quando bloquear:
  - criar draft `blocked`;
  - recomendar professor/manager;
  - opcionalmente criar task tecnica.

## Wave 4 - Kommo e Work Queue

- Se draft for seguro, permitir `Preparar na Kommo`.
- Se draft exigir professor, criar item Work Queue `personal_ai`.
- Work Queue deve mostrar:
  - badge `Personal IA`;
  - pergunta recebida;
  - orientacao sugerida;
  - evidencias usadas;
  - bloqueios;
  - CTA `Preparar na Kommo`, `Assumir`, `Encaminhar professor`, `Rejeitar`.

## Wave 5 - UI Coach Workspace / Perfil

- Adicionar bloco `Personal IA` no perfil operacional do aluno e/ou Coach Workspace.
- Professor pode gerar:
  - resumo tecnico do aluno;
  - resposta para duvida simples;
  - orientacao de rotina baseada no treino existente;
  - explicacao simples da avaliacao/bioimpedancia;
  - proxima acao tecnica.
- A UI sempre deve mostrar:
  - "IA assistiva; professor revisa antes do aluno receber".

## Wave 6 - Medicao e piloto

- Medir:
  - drafts gerados;
  - drafts aprovados;
  - drafts rejeitados;
  - bloqueios por safety;
  - tempo economizado por professor;
  - tarefas tecnicas criadas;
  - perguntas recorrentes.
- Validar com 20 casos reais:
  - 10 duvidas simples;
  - 5 explicacoes de avaliacao/bioimpedancia;
  - 5 casos sensiveis para escalacao.

## Criterio de sucesso

- Professor entende e valida uma resposta em menos de 30 segundos.
- Nenhum caso de dor/lesao/dieta/diagnostico gera resposta direta para aluno.
- Pelo menos 60% dos drafts simples sao aproveitaveis com pouca edicao.
- Toda orientacao cita evidencias reais do aluno.
