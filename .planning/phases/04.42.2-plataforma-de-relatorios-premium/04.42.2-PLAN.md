# 04.42.2 Plan

## Objetivo

Criar a fundacao tecnica e visual dos relatorios premium do AI GYM OS, substituindo a geracao textual atual
por um pipeline unico `HTML/CSS -> PDF` com payloads semanticos e componentes reutilizaveis.

## Escopo

1. Mapear e isolar os geradores atuais de PDF:
   - dashboards
   - bioimpedancia/composicao corporal
   - diagnostico
   - proposta
2. Criar um dominio unico de relatorio com:
   - `report_kind`
   - `report_scope`
   - `entity_id` ou `evaluation_id`
   - `parameters`
   - `generated_at`
   - `generated_by`
   - `version`
3. Implementar um renderer premium comum usando HTML/CSS e Playwright.
4. Criar a biblioteca-base de blocos premium e estilos compartilhados.
5. Definir contratos de payload separados dos dumps crus de dashboard.
6. Cobrir com testes de renderizacao, payload e compatibilidade com cache serializado.

## Fora de escopo

- white-label profundo por academia
- redesign de diagnostico/proposta como relatorio publico premium nesta rodada
- BI interativo novo

## Requisitos

- `RPT-01`: pipeline `HTML/CSS -> PDF`
- `RPT-02`: payload semantico por tipo
- `RPT-03`: biblioteca de blocos reutilizaveis
- `RPT-04`: branding leve da academia

## Gates de saida

- existe um renderer premium unico no backend
- existe um kit visual compartilhado para os relatorios
- os PDFs deixam de depender diretamente dos dumps textuais atuais
- a fundacao suporta os dois trilhos seguintes sem duplicacao estrutural
