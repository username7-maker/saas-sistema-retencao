# Feature Spec: Cordex Design System V1

## User Story

Como gestor, recepcionista, professor ou owner, quero que o Cordex Gym OS tenha identidade visual propria e confiavel, para perceber que estou usando um produto da Cordex e nao um template generico.

## Requisitos Funcionais

- `BRAND-DS-01`: O frontend deve usar tokens Cordex centralizados para cores, superficies e movimento.
- `BRAND-DS-02`: Login, sidebar e header devem usar assets Cordex adequados.
- `BRAND-DS-03`: Componentes base devem refletir o visual Cordex sem alterar seus contratos.
- `BRAND-DS-04`: Labels visiveis de IA devem usar linguagem Cordex quando fizer sentido.
- `BRAND-DS-05`: Relatorios premium devem usar Cordex como fornecedor/software e manter ProGym como academia.
- `BRAND-DS-06`: Rotas, tabelas, cookies, tokens, migrations e schemas tecnicos nao podem ser renomeados.

## Requisitos Nao Funcionais

- Build frontend deve permanecer verde.
- Acessibilidade minima de contraste deve ser preservada.
- Mudanca deve ser visual/comercial, sem impacto em autenticacao ou tenant scoping.
